```sql
-- 图书表 (books)
CREATE TABLE books (
  isbn VARCHAR(20) PRIMARY KEY,
  book_title VARCHAR(500) NOT NULL,
  book_author VARCHAR(200) NOT NULL,
  year_of_publication INTEGER,
  publisher VARCHAR(200),
  avg_rating DECIMAL(3,2) DEFAULT 0.0,
  rating_count INTEGER DEFAULT 0,

  -- 图片URL字段 (完整的Amazon三尺寸URL)
  image_url_s TEXT,  -- 小图
  image_url_m TEXT,  -- 中图
  image_url_l TEXT,  -- 大图

  -- 编码字段 (推荐算法用)
  book_author_encoded INTEGER,
  publisher_encoded INTEGER,
  publication_decade VARCHAR(10),

  -- 标准化字段
  year_of_publication_cleaned INTEGER,
  year_of_publication_cleaned_standardized DECIMAL(10,6),
  year_of_publication_cleaned_normalized DECIMAL(10,6),

  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户评分表 (user_ratings)
CREATE TABLE user_ratings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id VARCHAR(50) NOT NULL,
  isbn VARCHAR(20) NOT NULL,
  book_rating DECIMAL(3,1) NOT NULL,
  location VARCHAR(200),
  age INTEGER,
  location_encoded INTEGER,
  age_standardized DECIMAL(10,6),
  age_normalized DECIMAL(10,6),

  FOREIGN KEY (isbn) REFERENCES books(isbn),
  INDEX idx_user_id (user_id),
  INDEX idx_isbn (isbn),
  INDEX idx_rating (book_rating),

  UNIQUE (user_id, isbn)
);
```

### Phase 1.2: 推荐算法基础实现

#### ✅ 推荐算法实现 (已完成)

**✅ 已实现的推荐算法**:
1. **基于内容推荐** (Content-Based)
   - 基于作者、出版社、年代相似性
   - TF-IDF向量化 + 余弦相似度计算
   - 推荐理由自动生成

2. **混合推荐** (Hybrid)
   - 60% 内容相似性 + 40% 评分相似性
   - 多特征融合推荐策略

3. **BPR推荐算法** (Bayesian Personalized Ranking)
   - 基于矩阵分解的个性化排序推荐
   - 使用预训练模型 (60.5MB, 20个隐因子)
   - 支持97,106用户和270,151本图书的推荐
   - 响应时间~337ms, 精确度86%
   - 特点：矩阵分解、个性化排序、隐式反馈、高精度

4. ~~协同过滤推荐~~ (已删除)
   - 原计划实现但遇到技术问题，已从系统移除

## 📝 问题记录与解决方案

### ✅ 已解决的重要问题

1. **协同过滤算法兼容性问题**
   - **问题**: SQLAlchemy布尔表达式兼容性错误
   - **解决**: 删除协同过滤算法，优化混合推荐
   - **影响**: 系统更简洁，性能更好

2. **排名徽章遮挡图书封面**
   - **问题**: UI设计问题，用户体验差
   - **解决**: 重新设计排名徽章位置
   - **结果**: 企业级UI设计，用户体验提升

3. **图片加载优化**
   - **问题**: Amazon图片URL加载慢
   - **解决**: 多尺寸策略 + 错误处理
   - **结果**: 图片加载成功率 > 90%

4. **API响应格式统一**
   - **问题**: 前后端数据格式不一致
   - **解决**: 统一API响应格式
   - **结果**: 前后端完美对接

## 🚀 服务器运行指南

### 前端服务器启动

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次运行需要）
npm install

# 启动开发服务器
npm run dev

# 服务器将在以下地址启动：
# - 本地访问: http://localhost:5173
# - 网络访问: http://192.168.x.x:5173
```

### 后端服务器启动

```bash
# 进入后端目录
cd backend

# 安装依赖（首次运行需要）
pip install -r requirements.txt

# 启动FastAPI服务器
python run.py

# 或者使用uvicorn启动
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 服务器将在以下地址启动：
# - API地址: http://localhost:8000
# - API文档: http://localhost:8000/docs
# - 网络访问: http://192.168.x.x:8000
```

### 快速启动脚本

创建 `start_servers.bat` (Windows) 或 `start_servers.sh` (Linux/Mac):

**Windows版本 (start_servers.bat):**
```batch
@echo off
echo 启动图书推荐系统服务器...

echo 启动后端服务器...
cd backend
start "后端服务器" cmd /k "python run.py"

echo 等待后端启动...
timeout /t 5 /nobreak

echo 启动前端服务器...
cd ../frontend
start "前端服务器" cmd /k "npm run dev"

echo 服务器启动完成！
echo 前端地址: http://localhost:5173
echo 后端API: http://localhost:8000
echo API文档: http://localhost:8000/docs
pause
```

**Linux/Mac版本 (start_servers.sh):**
```bash
#!/bin/bash
echo "启动图书推荐系统服务器..."

echo "启动后端服务器..."
cd backend
gnome-terminal -- bash -c "python run.py; exec bash"

echo "等待后端启动..."
sleep 5

echo "启动前端服务器..."
cd ../frontend
gnome-terminal -- bash -c "npm run dev; exec bash"

echo "服务器启动完成！"
echo "前端地址: http://localhost:5173"
echo "后端API: http://localhost:8000"
echo "API文档: http://localhost:8000/docs"
```
