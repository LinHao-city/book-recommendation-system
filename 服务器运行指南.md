# 图书推荐系统 - 服务器运行指南

## 📋 环境要求

### 必需软件
- **Python**: 3.9+ (推荐 3.13.5，当前测试环境)
- **Node.js**: 16.0.0+ (推荐 22.20.0，当前测试环境)
- **npm**: 8.0.0+ (推荐 11.6.2，当前测试环境)

### 验证环境
```bash
# 检查 Python 版本
python --version
# 应显示: Python 3.13.5 或更高版本

# 检查 Node.js 版本
node --version
# 应显示: v22.20.0 或更高版本

# 检查 npm 版本
npm --version
# 应显示: 11.6.2 或更高版本
```

## 🚀 启动步骤

### 第一步：启动后端服务

1. **打开命令提示符或 PowerShell**

2. **进入后端目录**
   ```bash
   cd backend
   ```

3. **创建虚拟环境**（首次运行时需要）
   ```bash
   python -m venv venv
   ```

4. **激活虚拟环境**
   ```bash
   # Windows 命令提示符
   venv\Scripts\activate

   # Windows PowerShell
   .\venv\Scripts\Activate.ps1
   ```

5. **安装后端依赖**（首次运行时需要）
   ```bash
   pip install -r requirements.txt
   ```

6. **启动后端服务**

   **方式一：使用启动脚本（推荐）**
   ```bash
   python run.py
   ```

   **方式二：直接使用uvicorn**
   ```bash
   python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```

7. **验证后端启动**
   - 预期输出示例：
     ```
     INFO:     Will watch for changes in these directories: ['D:\\Projects\\图书推荐系统\\backend']
     INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
     INFO:     Started reloader process [10368] using WatchFiles
     ```
   - 浏览器访问: http://localhost:8000
   - API 文档: http://localhost:8000/docs

### 第二步：启动前端服务

1. **打开新的命令提示符窗口**

2. **进入前端目录**
   ```bash
   cd frontend
   ```

3. **安装前端依赖**（首次运行时需要）
   ```bash
   npm install
   ```

4. **启动前端服务**
   ```bash
   npm run dev
   ```
   **注意：** 默认端口为 5173，如需指定端口：
   ```bash
   npm run dev -- --port 5175 --host 0.0.0.0
   ```

5. **验证前端启动**
   - 预期输出示例：
     ```
     VITE v7.2.2 ready in 897 ms

     ➜  Local:   http://localhost:5173/
     ➜  Network: use --host to expose
     ```
   - 浏览器访问: http://localhost:5173 (或指定端口)

## 🛠 常用命令

```bash

```

## ⚠️ 注意事项

### 端口冲突
- 如果端口被占用，可以修改端口号：
  ```bash
  # 后端使用其他端口
  python -m uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload

  # 前端使用其他端口
  npm run dev -- --port 5176 --host 0.0.0.0
  ```

### 依赖问题
- 如果后端依赖安装失败：
  ```bash
  pip install --upgrade pip
  pip install -r requirements.txt
  ```

- 如果前端依赖安装失败：
  ```bash
  npm cache clean --force
  npm install
  ```

### 虚拟环境问题
- 如果虚拟环境激活失败，删除后重新创建：
  ```bash
  # 删除虚拟环境
  rmdir /s venv

  # 重新创建
  python -m venv venv
  ```

## 🔧 故障排除

### 问题 1: uvicorn模块未找到
**症状**: `No module named uvicorn`
**解决方案**:
```bash
# 安装uvicorn
pip install uvicorn

# 或者重新安装所有依赖
pip install -r requirements.txt
```

### 问题 2: 后端启动失败
**症状**: 端口 8000 无法访问
**解决方案**:
1. 检查 Python 和虚拟环境是否正确
2. 查看错误信息，可能是缺少依赖包
3. 重新安装依赖: `pip install -r requirements.txt`
4. 确认Python版本 >= 3.9

### 问题 3: 前端启动失败
**症状**: 端口 5173 (或指定端口) 无法访问
**解决方案**:
1. 检查 Node.js >= 16.0.0 和 npm >= 8.0.0 是否正确安装
2. 删除 `node_modules` 文件夹后重新 `npm install`
3. 检查端口是否被占用
4. 确认在frontend目录下执行命令

### 问题 4: 前后端连接失败
**症状**: 前端无法调用后端API
**解决方案**:
1. 确认后端服务正常运行
2. 检查 API 地址配置是否正确
3. 查看浏览器控制台的网络请求

### 问题 5: 数据库连接问题
**症状**: 后端日志显示数据库错误
**解决方案**:
1. 检查 SQLite 数据库文件是否存在
2. 确认数据库文件权限正确
3. 重新初始化数据库

### 问题 6: Python环境问题
**症状**: Python命令无法识别或版本不匹配
**解决方案**:
```bash
# 检查Python版本
python --version

# 如果版本过低，请升级Python到3.9+
# 确认使用正确Python环境
```

### 问题 7: 端口占用问题
**症状**: 端口8000或5173已被占用
**解决方案**:
```bash
# 查看端口占用
netstat -ano | findstr :8000
netstat -ano | findstr :5173

# 终止占用进程或更换端口
# 后端更换端口示例：
uvicorn app.main:app --port 8001

# 前端更换端口示例：
npm run dev -- --port 5174
```
